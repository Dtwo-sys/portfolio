# Calibrated Ideas Website Rebuild Brief

## 0. How to use this brief

This brief is the **source of truth** for the rebuild of **calibratedideas.com**. Read it in full before writing any code.

Two rules apply throughout:

1. **No invented specifications.** Do not fabricate dates, version numbers, technologies, metrics, testimonials, App Store URLs, or any factual claim. Where information is missing, leave the literal placeholder `[NEEDS INFO: …]` or `[TBD]` in the output and list it for review. Never guess.
2. **UK English everywhere**, in code comments, copy and metadata (optimise, colour, programme, behaviour, etc.).

A companion **`CLAUDE.md`** at the project root should restate these two rules plus the practical conventions (commit message style, package manager, lint/format commands, image-handling convention, "ask before publishing testimonials"). `CLAUDE.md` governs *how* to build; this brief governs *what* to build.

Anything in **Section 22 (Open questions)** must be resolved before the relevant work is treated as final.

---

## 1. Project overview

Rebuild **calibratedideas.com** as a striking, portfolio-led personal website for David / D2.

The current website presents David mainly as someone offering practical digital support, with sections for websites, marketing, technical writing, IT support and training. The new site should feel less like "I am looking for work" and more like "here is what I make, here is how I think, and here is the range of useful, thoughtful digital work I can produce".

The site should retain the warmth, plain-English tone and personal credibility of the existing site, but shift the emphasis towards:

- A curated portfolio of projects
- Case-study style storytelling
- Recent work and experiments
- Technical credibility without sounding corporate
- A distinctive visual identity
- A clear route for people to make contact, without the whole site feeling sales-led

The site will be rebuilt using a Node.js-based modern web stack and developed with Claude Code.

### Required migration script

Create a script such as:

```text

```

The script should:

   - <https://calibratedideas.com/>
   - <https://calibratedideas.com/about>
   - <https://calibratedideas.com/legal>

   If the current platform renders content with JavaScript, use a headless browser (Playwright) rather than a plain `fetch`.

2. Extract:
   - Page title
   - Headings
   - Paragraphs
   - Lists
   - Links
   - Image URLs
   - Image alt text

3. Save extracted content as Markdown or JSON:

```text
/imported/current-site/home.md
/imported/current-site/about.md
/imported/current-site/legal.md
/imported/current-site/assets.json
```

4. Download images into the working archive (not the public folder):

```text
/imported/current-site/images/
```

5. Preserve original filenames where possible.

6. Create a manifest:

```text
/imported/current-site/manifest.json
```

The manifest should include:

```json
{
  "sourceUrl": "https://calibratedideas.com/",
  "scrapedAt": "ISO timestamp",
  "pages": [],
  "images": [],
  "links": []
}
```

### Important notes

- Do not blindly reuse all current copy.
- Treat current text as source material, not final copy.
- Prioritise project descriptions, testimonials and biographical material.
- Remove repeated contact sections and duplicate navigation artefacts.
- Correct typographical errors during rewrite.
